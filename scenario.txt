#Сценарий ввода адреса доставки - Change for conflict from branch

**_Функциональные системные требования:_**

* Когда Пользователь задает адрес доставки, Система должна сохранить его. 
  
* Система позволяет Пользователю выбрать адрес только в городах и на улицах, входящих в зону доставки. 

**Цель:** ввести адрес доставки

**Акторы:** Пользователь, Система

##Основной сценарий:

1.	Пользователь заходит в корзину и переходит на шаг ввода адреса.
2.	Система отображает меню ввода адреса доставки. 
3.	Пользователь вводит имя и мобильный телефон. 
4.	Пользователь начинает вводить город доставки. 
5.	Система отображает Пользователю возможные значения, начинающиеся с введенных им букв.  
6.	Пользователь выбирает одно из значений. (См. также Альтернативный сценарий №1). 
7.	Пользователь начинает вводить улицу доставки. 
8.	Система отображает Пользователю возможные значения, начинающиеся с введенных им букв.  
9.	Пользователь выбирает одно из значений. (См. также Альтернативный сценарий №2). 
10.	Пользователь вручную вводит остальные сведения об адресе (дом, корпус, строение, подъезд, этаж, кв-ра, комментарий). 
11.	Пользователь переходит на следующий шаг: оплата.

##Альтернативный сценарий №1: 

Пользователь вводит название города вручную. 

1.	Пользователь вводит название города вручную.
2.	Система проверяет, что введенный город входит в зону доставки. Если город входит в зону доставки, то переход на следующий шаг, иначе завершение сценария. 
3.	Пользователь начинает вводить улицу доставки. 
4.	Система отображает Пользователю возможные значения, начинающиеся с введенных им букв.  
5.	Пользователь выбирает одно из значений. (См. также Альтернативный сценарий №2). 
6.	Пользователь вручную вводит остальные сведения об адресе (дом, корпус, строение, подъезд, этаж, кв-ра, комментарий). 
7.	Пользователь переходит на следующий шаг: оплата.


##Альтернативный сценарий №2: 

Пользователь вводит название улицы вручную. 

1.	Пользователь вводит название улицы вручную. 
2.	Система проверяет, что введенная улица входит в зону доставки. Если улица входит в зону доставки, то переход на следующий шаг, иначе завершение сценария. 
3.	Пользователь вручную вводит остальные сведения об адресе (дом, корпус, строение, подъезд, этаж, кв-ра, комментарий). 
4.	Пользователь переходит на следующий шаг: оплата.
